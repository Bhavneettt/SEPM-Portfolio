Software Engineering & Project Management

A semester-long journey into software development principles, agile practices, and real-world project execution

What I Learned

  • Software Development Life Cycles: Studied traditional models like Waterfall and modern approaches such as Agile, Scrum, and Extreme Programming.
  • Requirement Engineering: Learned how to gather user requirements, define functional and non-functional needs, and structure them into an SRS document.
  • System Design: Explored architectural design principles, patterns (MVC, layered), and tools like UML to visualize systems using Class, Use Case, Sequence, and State diagrams.
  • Effort Estimation & Planning: Applied COCOMO models and function point metrics to estimate cost, time, and resources required for projects. Built Gantt charts, WBS, and RACI matrices.
  • Testing & Quality Assurance: Understood black-box vs white-box testing, developed test cases, and executed unit, integration, system, and acceptance tests.
  • Risk & Release Management: Analyzed risks using proactive and reactive strategies. Learned about maintenance types (perfective, adaptive) and project closure activities.

Final Project: E Commerce Website

As part of the final semester project, our team developed E Commerce Website, a full-stack web platform to help people purchase, filter, sort, and update their products and carts.

We adopted Agile methodology, organizing our work into sprints and maintaining a product backlog. Daily stand-ups, sprint planning, and retrospectives ensured strong communication. I personally worked on:

  • Building responsive UI using HTML, CSS, and JavaScript
  • Contributing to the database schema and API integration
  • Documenting user stories, SRS, and test cases
  • Collaborating using GitHub, Trello, and Google Meet
